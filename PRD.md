## 2. 배경

기존 GPT 공유앱은 공용 생성형 AI 계정 사용 시간을 예약·관리하는 데 의미가 있었다. 그러나 생성형 AI 계정 로그인 자체를 외부 앱에서 통제하거나 실제 생성형 AI 서비스 사용량을 자동 조회하는 것은 현실적으로 어렵다.

따라서 이번 버전은 다음에 집중한다.

```text
공용 생성형 AI 계정 사용 예약
오늘 예약 현황 확인
사용 목적과 사용 기록 관리
프롬프트 기록 관리
OpenAI, Google Gemini, Anthropic Claude 사용자별 BYOK를 활용한 프롬프트 정리/개선
이용약관과 개인정보처리방침 제공
기관 대표번호 안내
```

즉, GPT를 직접 제어하는 앱이 아니라 **공용 AI 사용 운영을 기록하고 개선하는 앱**이다.

## 3. 프로젝트 목적

```text
교내 또는 팀 내 공용 AI 계정 사용을 예약·기록하고, 사용자가 작성한 거친 요청을 BYOK LLM Provider로 구조화된 프롬프트로 정리·개선하는 Flask 기반 웹앱을 만든다.
```

## 4. 연수 제출 목표

이번 프로젝트는 다음 연수 요구 요소를 충족한다.

```text
백엔드 사용
사용자 인증
Google OAuth
DB 저장
BYOK LLM API 활용
배포 가능성
교육 현장 문제 해결
```

다만 교육자료 자동 생성 앱은 만들지 않는다. 앱의 교육적 의미는 다음이다.

```text
교사가 생성형 AI를 안전하고 구조적으로 사용하도록 돕는다.
프롬프트 작성이 익숙하지 않은 사용자에게 역할, 맥락, 출력 형식, 검증, 반복 개선 개념을 자연스럽게 안내한다.
공용 AI 계정 사용 기록을 남겨 운영을 투명하게 한다.
```

## 7. 주요 사용자 시나리오

### 시나리오 A. 공용 AI 계정 예약

```text
1. 사용자가 로그인한다.
2. 대시보드와 오늘 예약 화면에서 예약 현황을 본다.
3. 사용할 AI 리소스를 선택한다.
4. 날짜, 시작 시각, 종료 시각, 사용 목적을 입력한다.
5. 서버가 예약 충돌을 검증한다.
6. 충돌이 없으면 예약이 저장된다.
7. 사용자는 내 예약 목록에서 예약을 확인한다.
```

### 시나리오 B. 사용 로그 작성

```text
1. 사용자가 예약 종료 후 사용 로그를 작성한다.
2. 작업 목적, 사용한 프롬프트, 결과 요약을 입력한다.
3. 로그는 사용자별로 저장된다.
4. 관리자는 전체 사용 흐름을 확인할 수 있다.
```

### 시나리오 C. 프롬프트 정리

```text
1. 사용자가 OpenAI, Google Gemini, Anthropic Claude 중 Provider를 선택한다.
2. API Key를 일회성으로 입력하거나 서버 암호화 저장을 선택한다.
3. 프롬프트 정리 메뉴로 이동한다.
4. 기존 요청과 정리 목표를 입력한다.
5. 서버가 정해진 구조화 프롬프트를 구성한다.
6. 선택한 Provider adapter가 사용자 본인 API Key로 모델을 호출한다.
7. 판정, 정리된 요청, 핵심 목표, 제약 조건, 검토 기준, 최종 프롬프트, 주의할 점이 표시된다.
8. 결과를 저장하고 복사할 수 있다.
```

### 시나리오 D. 운영 보고서 생성

```text
1. 관리자가 기간을 선택한다.
2. 앱이 예약/로그 데이터를 요약한다.
3. 운영 보고서 생성은 현재 BYOK 프롬프트 정리 범위 밖이며 필요 시 별도 결정한다.
4. 관리자는 보고서를 복사해 제출/공유할 수 있다.
```

이 기능은 시간이 부족하면 후순위로 둔다.

## 8. 프롬프트 정리기 요구사항

프롬프트 정리기는 자유 채팅이 아니다. 입력 폼 기반 도구다.

입력:

```text
작업 유형
원본 프롬프트
대상 사용자/맥락
원하는 출력 형식
개선 옵션 체크박스
```

개선 옵션:

```text
역할 부여
맥락 보강
제약 조건 명시
출력 형식 지정
단계별 수행
부족한 정보 질문
자기검토
반복 개선 루프
TDD/테스트 관점 추가
개인정보/보안 주의 추가
```

출력:

```text
판정: 채택 / 수정 후 채택 / 보류 / 폐기
정리된 요청
핵심 목표
제약 조건
검토 기준
최종 프롬프트
주의할 점
```

## 9. 비기능 요구사항

```text
3일 안에 완성 가능해야 한다.
로컬 Docker Compose로 실행 가능해야 한다.
OCI 단일 인스턴스에서 실행 가능해야 한다.
SQLite DB 파일은 ./instance에 저장해야 한다.
사용자 비밀번호와 API Key는 평문 저장하지 않는다.
LLM API Key는 프론트엔드로 재전송하거나 화면에 전체 표시하지 않는다.
테스트 코드가 있어야 한다.
README로 설치/실행/시연 방법을 설명해야 한다.
이용약관과 개인정보처리방침은 Markdown 파일로 관리하고 URL 공유 가능한 별도 페이지로 제공해야 한다.
```

