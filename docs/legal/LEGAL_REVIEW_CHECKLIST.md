# 법률 검토 체크리스트

이 문서는 서비스 운영 전 이용약관과 개인정보처리방침을 법률 전문가 또는 기관 개인정보보호 담당자가 검토할 때 확인할 항목입니다. 법률 자문 자체를 대체하지 않습니다.

## 검토 범위와 한계

```text
검토일: 2026-07-04
검토 성격: 코드/문서 정합성 및 개인정보 고지 항목 예비 점검
검토 한계: 변호사 법률 자문 또는 기관 공식 개인정보보호 검토가 아님
최종 필요 조치: 학교 또는 기관 개인정보보호 담당자, 필요 시 법률 전문가 검토
```

## 예비 검토 결과

```text
[확인] 이용약관과 실제 서비스 목적은 대체로 일치함
[수정] 신규 계정 승인 정책 문구를 실제 코드의 자동 승인 구조에 맞게 보정함
[수정] 개인정보처리방침 처리 항목에 부서, 내선번호, 인증번호 담당자 여부, 감사 로그 IP, 예약/로그/프롬프트 점검 세부 항목을 반영함
[수정] Gemini API 외부 전송 문구를 프롬프트와 점검 목표 전송 구조에 맞게 구체화함
[보완] 정보주체 권리, 파기 절차, 처리 위탁 확인 필요 항목을 추가함
[유지] 공용 생성형 AI 계정 ID/PW를 저장하지 않는다는 원칙은 약관과 방침에 유지함
```

## 이용약관 확인 항목

```text
[확인 필요] 서비스 목적과 실제 기능 범위가 일치하는가
[확인 필요] 공용 생성형 AI 계정 ID/PW를 저장하지 않는다는 설명이 명확한가
[확인 필요] 사용자가 입력하면 안 되는 개인정보/민감정보/평가 보안 자료가 명확한가
[확인 필요] 예약/사용 로그/프롬프트 점검 결과의 보관 목적과 책임 범위가 명확한가
[확인 필요] 서비스 중단, 계정 정지, 관리자 권한 행사 기준이 학교 운영 정책과 맞는가
[확인 필요] 신규 계정 자동 승인 정책이 실제 운영 정책으로 적절한가
```

## 개인정보처리방침 확인 항목

```text
[확인 필요] 수집 항목: 이메일, 이름, 부서, 내선번호, 비밀번호 해시, 권한, 승인 상태, 예약/사용 로그, API Key 암호문, 감사 로그 등
[확인 필요] 수집 목적: 사용자 인증, 예약 관리, 사용 이력 관리, 프롬프트 점검 기능 제공, 운영 감사
[확인 필요] 보관 기간과 삭제/파기 절차를 기관 기준으로 확정했는가
[확인 필요] 관리자/보조관리자의 접근 범위가 운영 권한 정책과 일치하는가
[확인 필요] Google OAuth 사용 시 외부 인증 제공자 정보 처리 설명이 충분한가
[확인 필요] Gemini API 호출 시 사용자의 개인 API Key 사용 구조와 외부 전송 설명이 충분한가
[확인 필요] 외부 서비스 이용이 제3자 제공, 국외 이전, 처리 위탁 중 어디에 해당하는지 기관 기준으로 판단했는가
[확인 필요] 정보주체 권리 행사 방법이 기관 내부 절차와 일치하는가
```

## 운영 전 확인

```text
[ ] ENABLE_REVIEW_ADMIN=false 확인
[ ] REVIEW_ADMIN_PASSWORD 운영 재사용 금지
[ ] ALLOWED_GOOGLE_DOMAIN 운영 도메인 정책 확인
[ ] 신규 계정 자동 승인 정책 유지 여부 확인
[ ] SESSION_COOKIE_SECURE=true 확인
[ ] WTF_CSRF_ENABLED=true 확인
[ ] SQLite 백업/복원 리허설 완료
[ ] 백업 파일 보관 기간과 폐기 절차 확정
[ ] Google OAuth와 Gemini API 외부 서비스 이용 고지 최종 확인
[ ] 기관 개인정보보호 담당자 최종 검토 완료
```
