# Security Decisions

## 1. GPT 계정 정보 저장 금지

결정:

```text
GPT 계정 ID/PW는 어떤 테이블에도 저장하지 않는다.
```

이유:

```text
계정 공유/접속 정보 관리는 보안·정책 리스크가 크다.
앱의 목적은 로그인 통제가 아니라 예약·운영 기록 관리다.
```

## 2. GPT 실제 사용량 자동 조회 제외

결정:

```text
ChatGPT 개인/공용 계정의 실제 메시지 사용량 자동 조회는 구현하지 않는다.
```

이유:

```text
외부 앱에서 개인 ChatGPT 계정 사용량을 안정적으로 조회하는 구조를 전제할 수 없다.
연수 제출용 범위를 벗어난다.
```

## 3. Gemini API Key 사용자별 암호화 저장

결정:

```text
사용자가 본인의 Gemini API Key를 설정한다.
서버는 Fernet으로 암호화해 DB에 저장한다.
암호화 키는 APP_ENCRYPTION_KEY 환경변수에 둔다.
```

주의:

```text
암호화 저장은 유출 위험을 줄이는 것이지 0으로 만드는 것이 아니다.
프론트엔드 노출, 로그 출력, 전체 키 재표시를 금지한다.
```

## 4. 자유 채팅형 UI 제외

결정:

```text
AI 기능은 필수적으로 프롬프트 점검기에 제한한다. 운영 보고서 생성은 시간이 남을 때만 구현하는 선택 기능이다.
```

이유:

```text
자유 채팅은 비용 통제가 어렵다.
프롬프트 인젝션과 엉뚱한 응답 위험이 크다.
연수 제출용 앱에서는 정형 워크플로우가 더 안정적이다.
```

## 5. 개인정보 입력 기능 제외

결정:

```text
학생 이름, 학번, 성적, 상담 내용, 생활기록부 자료 입력 기능을 만들지 않는다.
```

이유:

```text
개인정보 처리 승인·보관·파기·권한·로그 정책이 필요해 MVP 범위를 초과한다.
```
