# 종로산업정보학교 AI컴퓨터과 교육과정

[README로 돌아가기](../README.md)

## 1. 문서 목적

이 문서는 종로산업정보학교 AI컴퓨터과에서 왜 Python, Flask, SQLite, SQLAlchemy, Docker, Cloud 등을 순서대로 학습하는지 설명합니다. 단순한 기술 목록이 아니라 교육 철학, 기술 선정 이유, 프로젝트 운영 원칙, 생성형 AI 활용 원칙을 함께 정리합니다.

## 2. 교육 목표

```text
웹 서비스를 처음부터 끝까지 직접 개발할 수 있는 학생을 양성한다.
```

본 교육과정은 학생이 단순히 코드를 작성하는 데서 멈추지 않고, 전체 서비스의 구조를 이해하는 것을 목표로 합니다. 요구사항 분석부터 화면 설계, CRUD, 데이터베이스, 인증, 배포, 운영까지 실제 개발 과정 전체를 경험하도록 구성합니다.

이 프로젝트는 학교 공용 생성형 AI 계정 운영이라는 실제 학교 업무 문제를 다룹니다. 학생들은 추상적인 예제가 아니라 실제 현장 문제를 코드로 해결하는 과정을 볼 수 있습니다.

## 3. 교육 철학

가장 중요한 원칙은 다음과 같습니다.

```text
내가 이해하지 못하는 코드와 기술은 사용하지 않는다.
```

이 원칙은 다음 기준으로 운영합니다.

```text
새로운 기술보다 동작 원리를 먼저 이해한다.
생성형 AI가 작성한 코드도 이해하지 못하면 사용하지 않는다.
유지보수가 어려운 구조보다 단순한 구조를 우선한다.
최신 기술을 사용하는 것이 목표가 아니라 원리를 이해하고 문제를 해결하는 것이 목표이다.
학생이 프로젝트의 모든 코드를 직접 설명할 수 있는 수준을 목표로 한다.
```

좋은 프로젝트는 복잡한 기술을 많이 사용한 프로젝트가 아니라, 만든 사람이 구조와 이유를 설명할 수 있는 프로젝트입니다.

## 4. 생성형 AI 활용 원칙

본 교육과정은 생성형 AI를 적극 활용합니다. 다만 활용 시점을 구분합니다.

기초 학습 단계에서는 다음 핵심 개념을 학생이 직접 구현하도록 지도합니다.

```text
Python 문법
Flask 라우팅
Jinja2 템플릿
CRUD
SQLite 데이터베이스
SQLAlchemy 모델
```

8월 이후 개인 프로젝트 단계부터는 생성형 AI를 개발 파트너로 활용합니다. 이때 AI는 프로젝트를 대신 만들어 주는 역할이 아니라, 학생이 더 좋은 개발 과정을 경험하도록 돕는 협업 도구입니다.

AI 활용 범위:

```text
요구사항 정리
화면 설계 아이디어 검토
코드 리뷰
디버깅
리팩토링
테스트 작성
문서 작성
배포 절차 점검
```

학생은 AI가 생성한 코드도 직접 읽고 설명할 수 있어야 합니다. 설명할 수 없는 코드는 프로젝트에 넣지 않는 것을 원칙으로 합니다.

## 6. 프론트엔드 교육 원칙

수업에서 사용하는 프론트엔드 기술은 다음으로 제한합니다.

```text
HTML
CSS
Vanilla JavaScript
Tailwind CSS
```

개인 프로젝트도 동일한 규정을 적용합니다. React, Vue, Svelte, Angular 같은 SPA 프레임워크는 이 교육과정의 기본 범위에서 사용하지 않습니다.

그 이유는 SPA 프레임워크가 다음 개념을 충분히 이해한 이후 학습할 때 교육 효과가 높기 때문입니다.

```text
JavaScript 언어 이해
ES Module
비동기 처리
Component 기반 개발
Build Tool
TypeScript
```

현재 교육과정에서는 웹의 기본 동작 원리를 먼저 이해하는 것을 우선합니다. Tailwind CSS는 CSS 속성과 레이아웃 감각을 빠르게 연습하기 위한 도구로 사용합니다.

## 7. 백엔드 교육 원칙

교육과정에서는 Flask를 중심으로 웹 백엔드를 학습합니다. Flask를 선택한 이유는 다음 흐름을 비교적 적은 코드로 이해할 수 있기 때문입니다.

```text
HTTP
Routing
Template
CRUD
Database
Authentication
ORM
Deployment
```

Express.js, NestJS 같은 JavaScript 기반 백엔드는 좋은 도구이지만 JavaScript 언어 자체에 대한 높은 이해도가 필요합니다. 현재 교육과정에서는 Python으로 Flask를 학습하는 편이 웹 애플리케이션의 구조를 이해하는 데 더 적합하다고 판단했습니다.

## 8. 다른 프레임워크를 채택하지 않는 이유

다음 프레임워크는 모두 우수한 프레임워크입니다.

```text
Django
Django REST Framework
FastAPI
```

다만 본 교육과정에서는 교육 기간, 학생 수준, 학습 목표, 프로젝트 규모를 고려하여 Flask를 중심으로 운영합니다. FastAPI는 11~12월 소개 또는 심화 학습 주제로 다룰 수 있습니다. Django와 DRF도 이후 학습 방향으로 안내할 수 있습니다.
